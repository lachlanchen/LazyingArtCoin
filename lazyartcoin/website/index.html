Mainnet Launch Checklist

Follow these steps to deploy and activate LazyArtCoin on Ethereum mainnet.

  1. Fund the deployer wallet with ETH on mainnet (gas for deploy + verification + liquidity).
  2. Populate .env with MAINNET_RPC_URL, DEPLOYER_PRIVATE_KEY, OWNER_ADDRESS, and TREASURY_ADDRESS.
  3. Run npm install inside lazyartcoin/token to pull Hardhat and OpenZeppelin contracts.
  4. Deploy: npm run deploy:mainnet.
  5. Verify: npm run verify:mainnet -- <deployed_address> <owner> <treasury> <initial_supply>.
  6. Transfer ownership to the LazyArt multisig for operational safety.
  7. List LAC on a DEX (Uniswap) and seed liquidity with WETH.
  8. Publish your official contract address on lazy.art and socials.

FAQ

Answers to the most common questions contributors and fans ask.

How is LAC secured?

The core contract is powered by battle-tested OpenZeppelin libraries, supports emergency pausing, and should be administered by a Safe multisig. All upgrades are transparent on-chain.

Can supply expand over time?

Yes. The LazyArt treasury can mint new LAC for fresh campaigns or AI compute credits. All mint events are public so the community can track issuance.

What networks are supported?

Production runs on Ethereum mainnet. Use Sepolia for dry runs and QA by changing the Hardhat network flag.

Will there be staking or governance?

Future roadmap items include staking for power users and quadratic voting for featured creators. Join the LazyArt Discord to participate.
