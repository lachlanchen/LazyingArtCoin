Sepolia Launch Checklist

Gather these items and you can deploy test LAC in minutes.

  1. Grab Sepolia ETH from a faucet and fund your MetaMask test account.
  2. Create `.env` inside `lazyartcoin_test/token` with your Sepolia RPC URL, private key, owner, treasury, and optional initial supply.
  3. Run `npm install` inside `lazyartcoin_test/token` (first time only).
  4. Deploy via the web control center (recommended) or CLI: `npm run deploy:sepolia`.
  5. Add the new contract address to the dashboard when prompted.
  6. Mint LAC to teammates and have them import the token in MetaMask.
  7. Use `pause` and `unpause` to rehearse emergency runbooks.
  8. Document learnings before you move to mainnet.

Testnet FAQ

Everything you and your testers need to know.

Does test LAC behave like real LAC?

Yes. It uses the exact same contract, so all behaviors—from minting to pausing—match production.

How do testers get Sepolia ETH?

Direct them to a trusted faucet, send ETH to their MetaMask address, and have them connect to the control center.

Can we airdrop test LAC automatically?

Use the mint tools in the control center or write a short Hardhat script to distribute to many addresses at once.

What if we break something?

That’s the point of testing—deploy again on Sepolia, update the control center with the new contract address, and keep iterating.
